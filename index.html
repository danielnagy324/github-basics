GitHub Tutorial — Rövid jegyzet

Alapok, amiket rendszerüzemeltetőként érdemes tudni a Gitről és a GitHubról

Mi a Git, és mi a GitHub?

Rövid, közérthető definíciók:

  • Git — elosztott verziókezelő rendszer (Linus Torvalds fejlesztette 2005-ben). Kezeli a forráskód történetét, ágaztatást (branch), egyesítést (merge), stb.
  • GitHub — felhőalapú tároló és kollaborációs platform Git-re építve: távoli repository-k, pull requestek, issue-k, CI/CD integráció.

Gyors parancsok (alap munkamenet)

# helyi repo inicializálása
git init

# meglévő távoli repo klónozása
git clone git@github.com:felhasznalo/repo.git

# változások nyomon követése
git status
git add .
git commit -m "Rövid üzenet"

# feltöltés a távoli repoba
git push origin main

# változások lehúzása
git pull

# ág létrehozása és váltás
git checkout -b feature/uj-funkcio

Branching & Workflow

  • main/master — stabil, deploy-olható ág.
  • feature/* — új funkciók fejlesztése izoláltan.
  • hotfix/* — éles hibajavítások.
  • Pull request: kódreview + automatikus tesztek lefuttatása, mielőtt egyesítjük a változásokat.

SSH vs HTTPS

  • SSH — kulcspáros azonosítás, kényelmes és biztonságos (ajánlott fejlesztéshez).
  • HTTPS — egyszerű, de jelszó/token alapú hitelesítés kell (CI környezetben gyakori).

.gitignore és Release

  • .gitignore megakadályozza, hogy ideiglenes vagy bizalmas fájlok bekerüljenek a repóba (pl. .env, node_modules).
  • Tag-ek és release-ek: verziók kiadása (pl. v1.2.0). GitHub-on release changeloggal kombinálható.

Hasznos további parancsok

# ág lista
git branch -a

# más ág letöltése és átváltás
git fetch origin
git checkout origin/branch-nev -b local-branch

# rebase vs merge
git merge feature/abc
git rebase origin/main

# stash ideiglenes mentés
git stash
git stash pop

# commit visszaállítása
git revert

Rövid tipplista

Gyakorlati tanácsok üzemeltetőknek és fejlesztőknek

  • Használj SSH kulcsokat és 2FA-t GitHub fiókon.
  • Szegmentáld az IoT/edge és fejlesztői hálózatot — külön hálózati zóna a CI/CD és build-szerverek számára.
  • Automatizáld a build és teszt lépéseket (GitHub Actions).
  • Felderítéshez és auditokhoz engedélyezd a commit log és push naplózást.

Gyors fogalomtár

  • Repository (repo) — kód tárolója.
  • Fork — másolat egy másik fiók alá, jellemzően hozzájáruláshoz.
  • CI/CD — Continuous Integration / Continuous Delivery.
  • Actions — GitHub beépített CI eszköze.
